# Spotify Database Management System (DBMS Project)

This repository contains the database schema and EER model created for the Spotify music streaming platform as part of the DBMS assignment
---
## Project Contents

### 1. `spotify_streaming.sql`
SQL script containing the complete **database schema**:
- All tables  
- Attributes and data types  
- Primary keys  
- Foreign keys  
- Constraints  

### 2. `spotify_EER.pdf`
The **EER (Enhanced Entity Relationship) Diagram** representing all entities and relationships of the Spotify database.

### 3. `spotify_insert statements.sql`  
Contains a few basic sample `INSERT` queries to demonstrate how records can be inserted into the tables.  

### 4. `spotify_model.mwb`
The MySQL Workbench **model file** used to create and manage the EER diagram.  

---

## Project Overview

This project models the core functional components of Spotify, including:

- User management  
- Subscription plans & user subscriptions  
- Artists, albums, and tracks  
- Playlists and playlist-to-track mapping  
- Devices and listening sessions  
- Payments and billing  

The design follows relational database best practices with proper use of:
- Primary keys  
- Foreign key relationships  
- Normalized table structure  
- Appropriate data types  
- Referential integrity
